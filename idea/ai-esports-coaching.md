## 1. プロダクト概要

### コンセプト

Apex Legendsのプレイ動画をAI（マルチモーダルLLM）で解析し、**「上達の証明」** をコア価値とするサービス。

単なる「動画分析AI」ではなく、時系列での改善可視化（Improvement Graph）を軸にした **「上達体験プラットフォーム」** として設計する。

### Before → After（Codex + AgentTeamレビューによる修正）

| 項目 | Before（初期案） | After（v2） |
|------|------------------|-------------|
| コンセプト | 動画を解析してフィードバック | **上達体験プラットフォーム**（解析は機能の一つ） |
| 分析範囲 | ミクロ+マクロ全方位分析 | **マクロ分析特化**でスタート（ミクロは精度が出ない） |
| 訴求軸 | AIがすごい | **「ランクが止まる原因」を特定する** |
| 価格帯 | 月額2,480円 | **Lite 580円 / Pro 1,480円 / シーズンパス 5,980円** |
| キラー機能 | フィードバックレポート | **Improvement Graph（上達の因果グラフ）** |

### プロダクトの利点

1. いつでも好きな時にコーチングを受けられる
2. コーチングチケットの購入や、YouTubeの企画で運良く採用されるのを待つ必要がない
3. 時系列で「何を直した結果、何がどれだけ改善したか」を可視化できる

## 2. 解決する課題

### (a) 日程調整の難しさ

コーチ側との調整が大変。ゲーマーは日程調整や約束が苦手な人が多く、決まった時間に受講するのが難しい。

### (b) 既存サービスの普及率と不透明な取引

スキルタウンなどの既存サービスはあるものの普及率は低く、TwitterのDMで調整してPayPayで個人送金を行うなど、かなり無法地帯となっている。

### (c) 品質と価格の相関

良いコーチにお願いしようとすると価格が高くなり、主な層である高校生や大学生の手が出しづらい。人間コーチは1時間3,000-5,000円が相場。

| [STATUP.GG](https://statup.gg/) | LoLリアルタイム音声コーチ | リアルタイム性 | LoLのみ |

### 4.2 Razer Project AVAの脅威（レッドチーム詳細分析）

- **資本力**: 数十億円規模のR&D投資が可能
- **ハードウェアバンドル**: 「Razer製品を買えばAIコーチングが無料」モデルの可能性
- **データ量**: Synapse経由で数千万人のプレイデータ収集可能
- **ブランド力**: ゲーマーにとって信頼されたブランド
- **結論**: PMFを達成しても、Razerが同じ機能を無料で出せば市場を奪われるリスクがある

### 4.3 差別化戦略（Codexレビュー反映）

1. **改善証明型コーチング**: 単発フィードバックではなく、同一指標の時系列改善を可視化（例: 3週間で被弾前移動成功率+18%）
2. **Apex特化の意思決定フレーム**: リング読み、漁夫リスク、部隊構成相性をスコア化
3. **パーティ単位レビュー**: 3人の連携ミス（カバー遅延、フォーカス不一致、エントリー順）を自動検出
4. **日本語ネイティブ文脈**: 日本のプレイ文化・ランク帯・略語に合わせた指導
5. **「次の1試合でやること3つ」**: 長文解説ではなく行動変容に直結するチェックリスト

**注意（レッドチーム指摘）**: 「日本語ネイティブ」は防御壁にならない。ゲーマーは英語リテラシーが比較的高く、機械翻訳の品質も向上している。言語は参入障壁ではなく最低要件。

## 5. 技術的実現可能性

### 5.1 Gemini 2.5 Pro Video Understanding

| 項目 | スペック |
|------|---------|
| コンテキスト | 200万トークン（約6時間の動画処理可能） |
| FPS調整 | 低FPS（<1）で長動画、高FPSで高速アクション分析 |
| マルチモーダル | 映像+音声 |
| 出典 | [Google Developers Blog](https://developers.googleblog.com/en/gemini-2-5-video-understanding/) |

### 5.2 コスト試算（エンジニアレビュー反映）

**Gemini 2.5 Pro料金（15分動画の場合）:**

| 項目 | 計算 | コスト |
|------|------|--------|
| 入力トークン | 900秒 × 258トークン/秒 = 232,000トークン | $0.58 |
| 出力トークン | 約5,000トークン | $0.05 |
| **合計（全量投入）** | | **$0.63/動画** |

**コスト最適化策:**

| 施策 | 削減効果 | 実装難度 |
|------|----------|----------|
| FPSを0.5に落とす | 50%削減 | 低 |
| 一次処理でシーン絞り込み（30%に） | 70%削減 | 高 |
| Gemini Flash使用（Proの1/10） | 90%削減 | 低（精度低下あり） |
| バッチAPI使用（50%割引） | 50%削減 | 低（遅延許容必要） |
| キャッシュ: 同マップ・キャラの結果再利用 | 20-30%削減 | 中 |

**最適な組み合わせ**: ルールベース一次処理 → シーン絞り込み → Gemini Flash → 必要時Proにエスカレーション = **$0.05-0.15/動画**

### 5.3 技術的に実現可能なもの / 困難なもの

**実現可能（マクロ分析）:**
- ミニマップ読み・エリア移動判断 → Geminiが比較的得意
- キルフィード/キルログ検出 → テンプレートマッチング/OCR
- デスシーン検出 → 画面遷移のヒストグラム差分
- リングの縮小判断 → ミニマップの時系列分析

**精度不足（ミクロ分析）:**
- 体力ゲージの正確な数値読み取り → OCR/テンプレートマッチングの領域、LLMでは不安定
- AIM判定（クロスヘア追跡） → ピクセル単位の座標トラッキングが必要、LLMの得意領域外
- **銃声の方向・距離判定 → 音声波形のステレオ分析の問題。LLMではやるべきでない。初期スコープから除外すべき**

### 5.4 推奨アーキテクチャ（エンジニアレビュー反映）

**設計原則: LLMに「検出」と「解釈」を両方やらせない**

- **検出レイヤー**: 専用CV/MLパイプライン（ルールベース + YOLO等）
- **解釈レイヤー**: LLM（検出結果を人間にわかりやすく言語化）

```
[動画アップロード]
    │
    ├─→ [音声トラック自動除去] ← プライバシー対策
    │
    ▼
[一次処理: ルールベース + 軽量CV（LLM不使用）]
    ├─ キルフィード検出（テンプレートマッチング）
    │   └─ ゲームUIの位置は固定 → ハードコード可能
    ├─ デス検出（画面遷移ヒストグラム差分）
    ├─ ミニマップクロップ + フレーム間差分
    ├─ 音量スパイク検出（銃撃開始判定）
    └─ 出力: [(1:23, "kill"), (3:45, "death"), (5:12, "rotation_start")]
    │
    ▼
[二次処理: 重要シーンのみ]
    ├─ Gemini Flash で基本解析
    └─ 判断が複雑な箇所のみ Gemini Pro にエスカレーション
    │
    ▼
[レポート生成 + Improvement Graph更新]
    │
    ▼
[プッシュ通知 + シェアカード生成]
```

**本番アーキテクチャ:**

```
[ユーザー] → [CDN/S3アップロード]
                    │
                    ▼
            [メッセージキュー (SQS/Cloud Tasks)]
                    │
                    ▼
            [一次処理ワーカー群 (ECS/Cloud Run)]
                    │
                    ▼
            [二次処理キュー（レート制限付き）]
                    │
                    ▼
            [LLM解析ワーカー + リトライ/フォールバック]
                    │
                    ▼
            [レポート生成 + DB保存]
                    │
                    ▼
            [プッシュ通知でユーザーに完了通知]
```

**非同期処理は必須。** リアルタイムではなく「アップロードしたら後で通知」のUX。

### 5.5 API依存リスクと代替案

| リスク | 深刻度 | 説明 |
|--------|--------|------|
| API価格改定 | 極高 | Googleは予告なしに2-5倍にした前例あり |
| レート制限 | 高 | 200万トークンの大量リクエストはQPM/TPM制限に引っかかる |
| API仕様変更 | 高 | ビデオ理解は新機能であり不安定 |
| 精度のブラックボックス | 極高 | モデル更新で精度が急変する可能性 |

**対策**: マルチLLMアーキテクチャ（Gemini/GPT-4o間の切り替え可能な抽象化レイヤー）を初期設計から組み込む。

### 5.6 スケーラビリティの懸念

- 15分の1080p動画 ≈ 1-3GB。1,000ユーザーが毎日1本 = 1-3TB/日
- 一次処理5-10分 + LLM解析1-3分 + レポート生成30秒 = **最低10分の処理時間**
- この期待値ギャップをUXで吸収する必要あり

## 7. AgentTeamレビュー詳細

### 7.1 投資家レビュー

**判定: Conditional Pass（条件付き見送り）**

**評価された点:**
- 「上達の証明」というコア価値の設定。ゲーマーが欲しいのは分析レポートではなく「上手くなっている実感」

**指摘された構造的問題:**

1. **ユニットエコノミクスの破綻**: 旧価格設計（Lite 980円/Pro 2,480円）では全プランが赤字。API費用が月額を上回る
2. **単一タイトル依存**: Apexは成熟期。EA/Respawnのアップデート方針一つで市場が消える
3. **SOMとGo-to-Market戦略の不在**: 「誰に」「どうやって」届けるかが不明
4. **創業者のバックグラウンド未記載**: eスポーツ人脈、ML経験、ゲーマーとしての知見のどれがあるか

**投資条件（3つのうち2つを満たす必要あり）:**
- **条件A**: 1動画あたりAPI費用$0.3以下、LTV/CAC > 3、β30人が3ヶ月継続でチャーン10%以下
- **条件B**: WAU/MAU > 40%、フィードバック後の「次の試合プレイ率」80%以上
- **条件C**: Apex以外のタイトルへ2ヶ月以内に横展開できるアーキテクチャ実証

**Exit戦略:**
- 最も現実的: マルチタイトル展開後のゲーミング企業への売却（3-5年、10-30億円）
- 教育テック企業への売却も可能性あり（「動画ベースのスキル上達プラットフォーム」として再定義）

### 7.2 マーケターレビュー

**評価: 方向性○、実行解像度が粗い**

**主要な指摘:**
- ターゲティングが「年齢」で切っているが、ゲーマーは「ランク帯×プレイ姿勢」で行動が分かれる
- LP訴求「2週間で何が伸びるかを証明」は弱い。主語がサービス側で、ユーザーは「証明してもらう」ことに興味がない
- 「上達の証明」が「誰に対する証明か」が曖昧（自分自身 vs 友達/チームメイト）
- 無料→有料のコンバージョン設計がほぼ存在しない（最大の弱点）
- リテンション施策が一切書かれていない（重大な見落とし）
- 最大のリスク: 「良いプロダクトを作れば人が来る」という幻想

### 7.3 エンジニアレビュー

**評価: 設計の根本修正が必要**

**主要な指摘:**
- LLMに検出と解釈の両方をやらせるのは設計ミス
- 体力ゲージの数値読み取りはLLMでは不安定（OCR/テンプレートマッチングの領域）
- AIM判定はピクセル単位の座標トラッキングが必要で、LLMの得意領域外
- 銃声の方向・距離判定は「技術的に嘘に近い」。初期スコープから除外すべき
- 2段階解析の「一次処理」こそが技術的に最も難しく、ここの設計が曖昧なのは致命的
- **「計画を練る前にコードを書け。10本の動画を投げれば7割わかる」**

**Kill or Go判定項目（優先度順）:**
1. Geminiのゲーム画面理解精度が実用レベルか（キル/デス検出率90%以上）
2. フィードバックの質がプロコーチに匹敵するか（「役に立つ」評価60%以上）
3. Improvement Graphが意味のある変化を検出できるか
4. コストが月額課金モデルで回収可能か（粗利50%以上）

### 7.4 レッドチームレビュー

**失敗シナリオTOP5:**

| 順位 | シナリオ | 致命度 |
|------|---------|--------|
| 1 | Gemini API依存による事業存続不能（価格改定/仕様変更） | 10/10 |
| 2 | 解析の質が実際の上達に繋がらず、口コミで死ぬ | 9/10 |
| 3 | Apex Legends自体の衰退・大規模仕様変更 | 8/10 |
| 4 | 無料ティアがコンバージョンしない「フリーミアム地獄」 | 8/10 |
| 5 | 動画アップロードのUX障壁で初回体験離脱 | 7/10 |

**見落としている盲点:**
1. ゲーマーは「上達したい」と言いつつ「楽しみたい」（上達志向は市場の10-15%）
2. 動画を撮ってアップロードする層は既に自己分析能力がある（AIの付加価値が薄い）
3. 「日本語ネイティブ」は防御壁にならない
4. CAGR 17.5%は人間コーチング含む数字でAIコーチングの根拠にならない
5. 月次チャーン10%で12ヶ月後の残存は28%。ゲーマー向けは15-20%が一般的
6. 「Apex特化」がVC資金調達の足かせになる
7. 創業者自身がApexをやめたら終わる

**法的リスク:**
- EA/Respawnの利用規約違反リスク（ゲームプレイの商用利用にC&Dの可能性）
- 動画内の著作物（マップ、キャラ、UI）の商用インプット利用
- ボイスチャット音声のプライバシー問題
- 「上達の証明」の景品表示法リスク（優良誤認）

### 7.5 Codexレビュー

**核心的指摘:**
> 今の案のままだと「動画を解析してアドバイスするAI」は競合と同質化しやすく、価格競争になりやすい。勝ち筋は「分析の精度」よりも、**上達の証明**と**継続導線**をプロダクトに組み込めるか。

**キラー機能提案:**
- **Improvement Graph（上達の因果グラフ）**: 「何を直した結果、何がどれだけ改善したか」を自動で因果推定。単なる解説AIではなく「上達OS」になる。タイトル横展開時も競争優位が残る。
