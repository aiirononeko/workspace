# 記事ドラフト生成（microCMS）

承認済みの記事構成をもとにmicroCMSへ下書き記事を作成します。

**起動**: `/blog-draft <構成の説明またはキーワード>`（例: `/blog-draft DualShock4 スティック交換`）

## 前提

- `/blog-outline` で作成・承認された構成案が会話コンテキストにあること
- 構成案がない場合は、ユーザーにキーワードと概要を確認してから執筆
- microCMS MCPが設定済みであること

## 手順

### 1. microCMS MCPツールの準備

ToolSearchで `microcms` を検索し、利用可能なツールを確認する。

### 2. microCMSのAPI構造確認

microCMS MCPでコンテンツ一覧を取得し、APIのエンドポイント名（例: `blogs`, `articles`）とフィールド構造を把握する。

**初回は必ずこの確認を行うこと。** フィールド名はサービスごとに異なる。

### 3. ドラフト執筆

以下のルールに従って本文を作成:

**本文ルール**:
- 「です・ます」調で統一
- 一文は60文字以内を目安
- 専門用語は初出時に簡単な説明を付ける
- H2の直後に導入文を入れる（いきなりH3に入らない）
- 適度に箇条書き・表を使う
- 画像が必要な箇所に `<!-- IMAGE: 説明 -->` コメント（後で管理画面から追加）
- アフィリエイトリンク箇所に `<!-- AFFILIATE: 商品名 -->` コメント
- 体験を追記すべき箇所に `<!-- YOUR_EXPERIENCE: 説明 -->` コメント
- データが必要な箇所に `<!-- YOUR_DATA: 説明 -->` コメント

**文字数目安**:
- ハウツー記事: 3,000〜5,000字
- レビュー記事: 2,000〜4,000字
- 比較記事: 4,000〜6,000字

### 4. microCMSに下書き作成

microCMS MCPを使って下書きステータスで記事を作成する。

必須フィールド（APIの構造に合わせて調整）:
- タイトル
- 本文（HTML or リッチエディタ形式）
- メタディスクリプション
- カテゴリ / タグ（あれば）

**重要**: `status: "draft"` で作成すること（公開しない）。

### 5. 完了報告

```markdown
## ドラフト完成

📝 **microCMS**: 下書きとして作成済み
🔗 **管理画面**: microCMSの管理画面で確認・編集できます
📏 **文字数**: 約X,XXX字

### 要対応プレースホルダー

| # | 種類 | 箇所 | 内容 |
|---|------|------|------|
| 1 | IMAGE | H2: xxx | 管理画面から画像を追加 |
| 2 | YOUR_EXPERIENCE | H2: xxx | 実体験を追記 |
| 3 | AFFILIATE | H2: xxx | 商品リンクを挿入 |

### 次のステップ

1. microCMS管理画面で画像追加・プレースホルダー編集
2. `/blog-review` で品質チェック
3. `/blog-publish` で公開
```
