# 記事構成（アウトライン）生成

指定されたキーワードからSEO最適化された記事構成を作成します。

**起動**: `/blog-outline <キーワード>`（例: `/blog-outline DualShock4 スティック交換 方法`）

## 手順

### 1. 検索意図の分析

WebSearchでキーワードの検索結果を確認し、以下を特定:

- **検索意図**: 情報収集 / ハウツー / 比較検討 / 購入検討
- **ユーザー像**: どんな人がどんな状況で検索するか
- **求められている回答**: 具体的に何を知りたいか

### 2. 競合記事の構成分析

上位3〜5記事の見出し構成をWebFetchで取得し:

- 共通して含まれる見出し（必須トピック）
- 不足しているトピック（差別化チャンス）
- 記事の深さ・専門性レベル

### 3. 構成案の生成

以下のフォーマットで出力:

```markdown
# 記事構成案: [キーワード]

## 基本情報

- **ターゲットKW**: xxx
- **検索意図**: xxx
- **想定読者**: xxx
- **想定文字数**: X,000〜X,000字
- **カテゴリ**: 電子工作 / コーヒー / ソフトウェア / ガジェット

## タイトル案（3パターン）

1. 【ハウツー型】xxx
2. 【体験談型】xxx
3. 【網羅型】xxx

## メタディスクリプション（120文字以内）

> xxx

## 見出し構成

### H2: はじめに / リード文
- 読者の悩みに共感
- この記事で解決できること
- 筆者の経験・信頼性

### H2: [本題1]
#### H3: xxx
- ポイント
#### H3: xxx
- ポイント

### H2: [本題2]
...

### H2: まとめ
- 要点の振り返り
- 次のアクション提案

## 差別化ポイント

- 独自の体験・データを入れるべき箇所
- 写真・図解が必要な箇所
- アフィリエイトリンクの自然な挿入ポイント

## 内部リンク候補

- 関連しそうな記事テーマ
```

### 4. 対話

構成案の表示後、以下を提案:

- 構成の修正・調整
- タイトル案の選択
- 承認して `/blog-draft` でドラフト作成に進む
