# 記事品質レビュー

ブログ記事のドラフトを4軸で採点し、具体的な改善案を提示します。
microCMS上の記事、またはローカルファイルの両方に対応。

**起動**:
- `/blog-review <記事タイトルまたはキーワード>` — microCMSから記事を検索してレビュー
- `/blog-review <ファイルパス>` — ローカルファイルをレビュー（フォールバック）

## 手順

### 1. 記事の読み込み

**microCMSの場合**:
1. ToolSearchで `microcms get` を検索し、`mcp__microcms__microcms_get_list` と `mcp__microcms__microcms_get_content` をロード
2. `mcp__microcms__microcms_get_list`（endpoint: `blogs`）で記事一覧を取得
3. タイトルやキーワードで該当記事を特定
4. `mcp__microcms__microcms_get_content`（endpoint: `blogs`, contentId: `...`）で本文を取得
5. 取得できるフィールド: `title`, `content`（HTML）, `category`

**ローカルファイルの場合**:
- 指定されたファイルを読み込む
- front matterからメタ情報を抽出

### 2. SEOチェック（30点満点）

| チェック項目 | 配点 | 基準 |
|-------------|------|------|
| タイトルにメインKW含有 | 5点 | タイトルの前半にKWがあるか |
| メタディスクリプション | 5点 | 120文字以内、KW含有、行動喚起 |
| H2/H3にKW自然に含有 | 5点 | 不自然な詰め込みでないか |
| 見出し階層の正しさ | 5点 | H2→H3の順序、飛ばしなし |
| 内部リンク候補の有無 | 3点 | 関連記事への誘導があるか |
| 画像alt候補の有無 | 3点 | 画像にalt説明があるか |
| URL slug の最適化 | 2点 | 短く、KWを含み、英語 |
| 構造化データ対応 | 2点 | FAQ、HowToなどの対応余地 |

### 3. コンテンツ品質チェック（30点満点）

| チェック項目 | 配点 | 基準 |
|-------------|------|------|
| 検索意図との一致 | 8点 | WebSearchで確認した検索意図に応えているか |
| 独自性・体験価値 | 8点 | 実体験やオリジナルデータが含まれるか |
| 情報の正確性 | 5点 | 技術的に誤った記述がないか |
| 網羅性 | 4点 | 競合記事と比較して不足がないか |
| CTA・収益導線 | 3点 | アフィリエイトリンクの自然さ |
| 差別化要素 | 2点 | 他記事にない独自の切り口 |

### 4. 読みやすさチェック（15点満点）

| チェック項目 | 配点 | 基準 |
|-------------|------|------|
| 文の長さ | 4点 | 60文字以内が目安 |
| 段落の長さ | 4点 | 3〜4文で改行 |
| 箇条書き・表の活用 | 4点 | 適度に使われているか |
| 文体の統一 | 3点 | です/ます調の統一 |

### 5. AI臭チェック（25点満点）

AI生成文章特有の不自然さがないかを厳しくチェックする。
`/blog-humanize` を経由していても、残存するAI臭がないか最終確認する。

| チェック項目 | 配点 | 基準 |
|-------------|------|------|
| 記号の残存 | 4点 | `**太字**`の残り、「—」emダッシュ、「：半角スペース」、／並列がないか |
| 語尾のバリエーション | 4点 | 同じ語尾が3回以上連続していないか。「〜です。〜です。〜です。」の単調さ |
| 接続詞の過多 | 3点 | 「さらに」「また」「したがって」「そのため」「結果として」の連打がないか |
| 前置き・宣言の説明書感 | 3点 | 「本記事では」「以下で解説します」「3つの観点から」のような構造宣言がないか |
| 事なかれ主義の表現 | 3点 | 「一概には言えませんが」「状況によって異なります」「メリットもあればデメリットも」がないか |
| 抽象語・万能語の空回り | 3点 | 「本質」「最適化」「価値を最大化」「非常に有効」が根拠なく使われていないか |
| テンプレ比喩 | 2点 | 羅針盤、設計図、土台、エンジン、スパイス等のありきたり比喩がないか |
| カタダ文体との一致 | 3点 | カタダらしい語尾・トーン・表現が使われているか（下記参照） |

**カタダ文体チェックポイント**:
- 冒頭に軽い名乗りがあるか
- 「〜ですよね」「〜なんですよね」「〜かなと」など口語的語尾が混ざっているか
- 熱い箇所と軽い箇所で温度差があるか
- スタンスを取っているか（保険で逃げていないか）
- 「参考になれば幸いです」等のテンプレ締めになっていないか
- 個人的な感想・主観が適度に入っているか

**AI臭の具体的なNG表現リスト**（検出したら減点）:
- 「本質を押さえ」「価値を最大化」「最適化する」
- 「持続可能な」「多角的な視点」
- 「架け橋となる」「エコシステム」「バリュー」
- 「非常に重要です」（根拠なし）
- 「以上がポイントです。次に進みましょう。」
- 「ステップ1：」「STEP」
- 「状況によって異なります」「ケースバイケースです」
- 「まずは小さく始めてみましょう」
- 「参考になれば幸いです」
- 「一概には言えませんが、一般的には〜」
- 「あまり推奨されません」「注意が必要です」（言い切れる場面）

### 7. 対話

レビュー後、以下を提案:
- AI臭が多い場合: `/blog-humanize` での再変換を推奨
- 指摘箇所の自動修正（microCMS MCPで更新可能なもの）
- 修正後の再レビュー
- 特定カテゴリの深掘りレビュー
