# 記事品質レビュー

ブログ記事のドラフトを3軸で採点し、具体的な改善案を提示します。
microCMS上の記事、またはローカルファイルの両方に対応。

**起動**:
- `/blog-review <記事タイトルまたはキーワード>` — microCMSから記事を検索してレビュー
- `/blog-review <ファイルパス>` — ローカルファイルをレビュー（フォールバック）

## 手順

### 1. 記事の読み込み

**microCMSの場合**:
1. ToolSearchで `microcms` を検索し、利用可能なツールを確認
2. コンテンツ一覧を取得してAPIのエンドポイント名を把握
3. 記事タイトルやキーワードで検索し、該当記事を取得
4. 本文・タイトル・メタディスクリプション等を取得

**ローカルファイルの場合**:
- 指定されたファイルを読み込む
- front matterからメタ情報を抽出

### 2. SEOチェック（40点満点）

| チェック項目 | 配点 | 基準 |
|-------------|------|------|
| タイトルにメインKW含有 | 5点 | タイトルの前半にKWがあるか |
| メタディスクリプション | 5点 | 120文字以内、KW含有、行動喚起 |
| H2/H3にKW自然に含有 | 5点 | 不自然な詰め込みでないか |
| 見出し階層の正しさ | 5点 | H2→H3の順序、飛ばしなし |
| 内部リンク候補の有無 | 5点 | 関連記事への誘導があるか |
| 画像alt候補の有無 | 5点 | 画像にalt説明があるか |
| URL slug の最適化 | 5点 | 短く、KWを含み、英語 |
| 構造化データ対応 | 5点 | FAQ、HowToなどの対応余地 |

### 3. コンテンツ品質チェック（40点満点）

| チェック項目 | 配点 | 基準 |
|-------------|------|------|
| 検索意図との一致 | 10点 | WebSearchで確認した検索意図に応えているか |
| 独自性・体験価値 | 10点 | 実体験やオリジナルデータが含まれるか |
| 情報の正確性 | 5点 | 技術的に誤った記述がないか |
| 網羅性 | 5点 | 競合記事と比較して不足がないか |
| CTA・収益導線 | 5点 | アフィリエイトリンクの自然さ |
| 差別化要素 | 5点 | 他記事にない独自の切り口 |

### 4. 読みやすさチェック（20点満点）

| チェック項目 | 配点 | 基準 |
|-------------|------|------|
| 文の長さ | 5点 | 60文字以内が目安 |
| 段落の長さ | 5点 | 3〜4文で改行 |
| 箇条書き・表の活用 | 5点 | 適度に使われているか |
| 文体の統一 | 5点 | です/ます調の統一 |

### 5. 出力フォーマット

```markdown

📍 **ソース**: microCMS下書き / ローカルファイル
📅 **レビュー日**: YYYY-MM-DD

## 総合スコア: XX / 100点

| カテゴリ | スコア | 判定 |
|---------|--------|------|

| コンテンツ品質 | XX/40 | ⭐⭐⭐ |

### 判定基準
- 80点以上: ✅ 公開OK
- 60-79点: ⚠️ 修正推奨
- 60点未満: ❌ 要リライト

## 改善項目（優先度順）

1. **[カテゴリ] 項目名**
   - 現状: xxx
   - 改善案: xxx
   - 影響: スコア+X点

### 🟡 推奨修正

1. **[カテゴリ] 項目名**
   - 現状: xxx
   - 改善案: xxx

### 🟢 あると良い

1. **[カテゴリ] 項目名**
   - 提案: xxx

## 良い点

- xxx
- xxx

## 次のステップ

- 80点以上: `/blog-publish` で公開
- 80点未満: 修正後に再度 `/blog-review` を実行

```

### 6. 対話

レビュー後、以下を提案:
- 指摘箇所の自動修正（microCMS MCPで更新可能なもの）
- 修正後の再レビュー
- 特定カテゴリの深掘りレビュー
